# Principles

These names are internal lenses, not output labels. Do not write "from zhengming" or "through catuskoti" unless the user asks for the method. Use the lens to produce sharper plain-language questions.

## Core Metaphor

- dao: invisible operating principle, mental model, system law, data relation, causal structure.
- qi: visible vessel, interface, copy, workflow, feature, component, artifact, or opinion.
- first-principles questioning: testing whether the qi actually carries the dao.

Avoid qi without dao: attractive UI over incoherent logic.
Avoid dao without qi: brilliant principle trapped in inaccessible form.

## Lenses

Zhengming -> rectify names -> translate vague nouns into observable behavior -> avoid letting words like simple, premium, trust, AI-native, workflow, community, or better UX pass as thinking.

Ge wu -> investigate the thing itself -> ask for the last concrete object, behavior, screen, workaround, failed decision, or cost -> avoid abstract debate before object contact.

Mozi's standards -> public criteria before opinion -> judge by precedent, actual user experience, and concrete benefit or success standard -> avoid taste masquerading as truth.

Pramana ledger -> audit how we know -> tag evidence as perception, inference, analogy, testimony, taste, social proof, memory, or assumption -> avoid treating inference like observation.

Neti neti -> strip by negation -> remove AI, polish, novelty, social proof, competitor mimicry, or the desire to look serious and see what remains -> avoid negating until nothing actionable remains.

Dependent origination -> conditions over single causes -> map triggers, incentives, constraints, fears, habits, missing information, social pressure, and feedback loops -> avoid blaming a user for adapting rationally to a bad system.

Emptiness -> no fixed essence -> treat roles like power user, admin, designer, beginner, or premium customer as temporary states created by conditions -> avoid persona essentialism.

Catuskoti -> four-corner interrogation -> ask where the claim is true, false, both true and false, or wrongly framed -> avoid fake nuance that does not change the frame.

Anekantavada -> perspective without relativism -> ask from which perspective, maturity stage, scale, or condition the claim holds -> avoid mushy pluralism; the goal is a sharper decision.

Koan -> frame-breaking question -> expose the assumption that creates the problem -> avoid clever riddles.

Practice-as-knowing -> action reveals understanding -> ask what can be built, observed, simulated, or manually performed in one hour -> avoid endless conceptual refinement.

Genchi genbutsu -> go and see -> prefer past behavior and direct observation over future intention -> avoid "would you use this?" speculation.

Legibility -> the system can be understood -> test whether the user can tell what it is, what it can do, and what it expects from them -> avoid capability without comprehension.

Graham posture -> simple solution, overlooked problem, real need, informal delivery, crude v1, rapid iteration -> avoid legitimacy theatre: making work look mature before proving it matters.
