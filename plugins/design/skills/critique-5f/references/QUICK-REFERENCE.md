# UX Reviewer - Quick Reference Card

One-page cheat sheet for using Saurabh Soni's design critique agent.

---

## 🚀 Quick Start

1. Upload design screens (screenshots, Figma links, videos)
2. Say: **"Review this design"** or **"Give me UX feedback"**
3. Answer 7 clarifying questions
4. Choose analysis type and mood
5. Get actionable feedback

---

## 📋 The 7 Clarifying Questions

| # | Question | What to Provide |
|---|----------|-----------------|
| 1 | **Screen Context** | Purpose, what user did before, what's next |
| 2 | **Target User** | Role, technical level, motivations |
| 3 | **Job-to-be-Done** | The key task to accomplish |
| 4 | **Ideal Outcome** | Success criteria from user's perspective |
| 5 | **Product Context** | What the product does, screen's function |
| 6 | **Design Stage** | Exploration / Experimentation / Final UI + flow type |

## 📊 Scoring Scale

| Score | Meaning | Action |
|-------|---------|--------|
| **1** | Critical flaw—actively harms UX | Urgent redesign required |
| **2** | Major improvement needed | Significant gaps to address |

## 💡 Pro Tips

### ✅ Do This
- **Be specific** → "Operations managers monitoring API uptime" > "users"
- **Upload quality screens** → High-res, complete flows
- **Admit unknowns** → "I haven't decided target proficiency yet"
- **Request both analyses** → Most comprehensive feedback
- **Use Round 2** → For complex multi-step flows

### ❌ Avoid This
- Vague one-word answers to clarifying questions
- Blurry or incomplete screenshots
- Skipping design stage specification
- Expecting analysis without context
- Taking critique personally

## 📏 The 10 Heuristics (Quick Reminder)

1. **Visibility of System Status** → User always knows what's happening
2. **Match System/Real World** → Familiar terms, logical structure
3. **User Control & Freedom** → Easy undo, clear exits
4. **Consistency & Standards** → Predictable patterns
5. **Error Prevention** → Proactive constraints, validation
6. **Recognition vs. Recall** → Self-evident, low cognitive load
7. **Flexibility & Efficiency** → Shortcuts for power users
8. **Aesthetic & Minimalist** → Every element serves purpose
9. **Help Users Recover** → Clear, constructive error messages
10. **Help & Documentation** → Contextual, accessible guidance

---

## 🏆 The 5 Strategic Pillars

1. **Strategic Alignment** → Problem-solution fit, JTBD simplicity, business goal harmony
2. **Usability & Journey** → Task efficiency, intuitive flow, end-to-end cohesion
3. **Visual Design & Clarity** → Hierarchy, professionalism, brand alignment
4. **Content & Interaction** → Copy clarity, interaction feedback
5. **Innovation & Delight** → Competitive edge, delightful details

## 🇮🇳 The 5F Principles (B2B SaaS)

**🆕 Research-backed framework from Razorpay for Indian B2B SaaS market**

### F1: Make it Fast (Convenience)
- **Proactive Feedback** → Warn before errors, not after
- **Meet Users Where They Are** → Integrate with tools users already use (Slack, WhatsApp, Teams, email), in-dashboard help
- **Forgiving Design** → Clear escape routes, editable fields

### F2: Make it Focused (Attention)
- **Information Hierarchy** → Glance (10s metrics) >> Act (action zone) >> Explore (deep dive)
- **Fight Banner Blindness** → No cross-sells in prime space
- **Feature Paralysis** → Show 3-5 role-relevant tools, hide rest
- **Cultural Relevance** → Local icons, Hinglish, social proof tags

### F3: Make it Fun (Delight)
- **Thought Speed** → Predictive defaults, smooth animations, cognitive resume
- **Dopamine Hits** → Micro-interactions, celebration states, meaningful empty states
- **Business Impact** → Show ROI, humanize metrics
- **Dashboard Personality** → Operational anchor, performance enabler, or autonomous ideal

### F4: Make it Fluent (Learnability)
- **Intuitive** → Learning + Efficiency + Memory
- **Persona Customization** → CEO view vs admin view vs ops view
- **Navigation Memory** → Macro-to-micro, breadcrumbs, lite version
- **RBAC** → Multi-level Indian hierarchy support

### F5: Make it Fair (Trust)
- **Emotional Intelligence** → Anticipate needs, empathetic copy
- **Radical Transparency** → Micro-step progress tracking, data-rich UI
- **Business Language** → Outcomes over jargon, contextual Hinglish
- **AI Trust** → Confidence scores, source of truth, accountability

**Full details:** See `5F-FRAMEWORK-REFERENCE.md`

## 🚨 When to Use Each Framework

| Situation | Use Framework | Why |
|-----------|---------------|-----|
| Quick UX audit | Heuristic (Nielsen) | Universal standards, fast |

| B2B SaaS (India market) | 5F Framework | Culturally relevant, evidence-based |
| Payment/fintech product | Heuristic + 5F | Trust + usability critical |

| Pre-launch readiness | Strategic Pillars 1-4 | Alignment, usability, visual, content |
| Competitive differentiation | Strategic Pillar 5 + 5F F3 | Innovation + delight |
| User adoption struggling | 5F F1, F4 + Strategic Pillar 2 | Fast, fluent, journey |
| Trust issues | 5F F5 | Radical transparency, EQ |

## 🎓 Learning from Feedback

### Understanding Low Scores (1-2)
- **Don't panic** → These are opportunities
- **Focus on impact** → Fix highest-impact issues first
- **Ask follow-ups** → "How would you approach fixing X?"

### Leveraging High Scores (4-5)
- **Document patterns** → What made this work?
- **Apply elsewhere** → Replicate success in other screens
- **Share with team** → Celebrate wins

### Acting on Recommendations
- **Prioritize by effort/impact** → Quick wins first
- **Tie to metrics** → Track before/after
- **Iterate** → Re-submit after fixes for validation

## 💪 Making the Most of Soni

### Before the Session
- Gather context (user research, business goals)
- Ensure screens are complete/high-quality
- Know your design stage
- Identify specific concerns if any

### During the Session
- Answer clarifying questions thoroughly
- Be honest about unknowns
- Take notes on scores and justifications
- Ask follow-up questions

### After the Session
- Prioritize recommendations by impact
- Share scorecards with team
- Create action items from feedback
- Schedule re-review after fixes
