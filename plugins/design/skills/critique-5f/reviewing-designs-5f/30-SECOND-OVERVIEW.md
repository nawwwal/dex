# 5F Learning System - 30-Second Overview

```
┌─────────────────────────────────────────────────────────────────┐
│  WHAT IT DOES                                                   │
│  Your design reviewer learns YOUR context and gets smarter      │
└─────────────────────────────────────────────────────────────────┘

   You use it → Answer quick questions → Give optional feedback
                            ↓
                    System learns:
                    • Your users
                    • Your business rules
                    • Your quality bar
                    • Your design patterns
                            ↓
                Next review: More accurate
                    (repeat forever)


┌─────────────────────────────────────────────────────────────────┐
│  HOW TO USE IT                                                  │
└─────────────────────────────────────────────────────────────────┘

  1. Run your review normally
  2. See a question? → Answer it (15 sec)
  3. See feedback prompt? → Rate + comment (10 sec)
  4. Done. System learned.

  First 5 reviews: Answer questions (builds 80% of context)
  Reviews 6-20:    Validate patterns (system learns your standards)
  Reviews 21+:     Maintenance mode (minimal input needed)


┌─────────────────────────────────────────────────────────────────┐
│  WHAT YOU GET                                                   │
└─────────────────────────────────────────────────────────────────┘

  ✓ Auto-checks YOUR business rules (compliance, regulations)
  ✓ Scores for YOUR users (not generic best practices)
  ✓ Uses YOUR quality thresholds (learns from your overrides)
  ✓ Recognizes YOUR design patterns (standard vs. deviation)

  Review #1:  Generic scoring, many questions
  Review #10: Context-aware, fewer questions
  Review #30: Near-perfect accuracy, minimal questions


┌─────────────────────────────────────────────────────────────────┐
│  COMMANDS                                                       │
└─────────────────────────────────────────────────────────────────┘

  /context-5f        → What was learned?
  /context-5f edit   → Fix wrong learnings
  /context-5f report → Monthly summary


┌─────────────────────────────────────────────────────────────────┐
│  TIME INVESTMENT                                                │
└─────────────────────────────────────────────────────────────────┘

  First 5 reviews:  5 min each = 25 min total → 80% context built
  Reviews 6-20:     3 min each = 45 min total → Patterns validated
  Reviews 21+:      2 min each → Near-perfect accuracy

  Total: ~2 hours over first month
  Payoff: Permanently smarter reviews forever


┌─────────────────────────────────────────────────────────────────┐
│  KEY PRINCIPLE                                                  │
└─────────────────────────────────────────────────────────────────┘

  The more specific your feedback, the smarter the system gets.

  ❌ "Missed compliance"
  ✅ "RBI requires explicit consent for PAN collection"

  One good answer = Auto-checked in all future reviews


┌─────────────────────────────────────────────────────────────────┐
│  GET STARTED                                                    │
└─────────────────────────────────────────────────────────────────┘

  Just run: reviewing-designs-5f [your-design]

  Answer the first question you see.
  That's it.

  System handles everything else automatically.


┌─────────────────────────────────────────────────────────────────┐
│  LEARN MORE                                                     │
└─────────────────────────────────────────────────────────────────┘

  • CHEAT-SHEET.md     → Quick reference (1 min read)
  • HOW-TO-USE.md      → Best practices (5 min read)
  • QUICKSTART.md      → Getting started guide (2 min read)
  • README.md          → Full documentation (20 min read)
```
