# Persuasive Design

Use persuasive design ethically. The goal is to help users do what they already have reason to do, not trap them.

## Fogg Behavior Model

Behavior happens when:

- Motivation exists
- Ability is sufficient
- Prompt arrives at the right moment

For every persuasive design idea, specify:

- Desired behavior:
- Motivation:
- Ability/friction:
- Prompt:
- Timing:
- User benefit:
- Business benefit:
- Ethical boundary:
- Opt-out or escape:
- Failure mode:

## Persuasive levers

- Reduce steps
- Pre-fill safely
- Show progress
- Show consequence
- Use defaults
- Use reminders
- Use social proof
- Use commitment
- Use comparison
- Use urgency
- Use scarcity only if real
- Use reward only if earned

## Anti-patterns

Never:

- Hide cost
- Hide consequences
- Make cancellation hard
- Use fake scarcity
- Use fake urgency
- Shame users
- Use streaks for serious obligations
- Make destructive actions too easy
- Make safe actions unnecessarily hard
- Use confirmshaming
- Confuse users into consent
