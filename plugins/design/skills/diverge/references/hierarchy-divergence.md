# Information Hierarchy Divergence

Use when the question is what should be most prominent, what should be quiet, and what should disappear.

## Hierarchy inputs

- User decision
- User mental state
- Time pressure
- Risk
- Frequency of use
- Expertise
- Business priority
- State severity
- Data freshness
- Trust requirement

## Hierarchy strategies

- Outcome-first
- Action-first
- Risk-first
- Status-first
- Change-first
- Recommendation-first
- Comparison-first
- Timeline-first
- Exception-first
- Owner-first
- Evidence-first

## Hierarchy output

For every direction, define:

1. First read:
2. Second read:
3. Third read:
4. Persistent context:
5. Hidden detail:
6. Quiet metadata:
7. Removed content:

## Squint test

Perform this description:

```md
With blurred vision, the user should first see <X>, then <Y>, then <Z>.
```

If the squint test says "everything", hierarchy failed.
