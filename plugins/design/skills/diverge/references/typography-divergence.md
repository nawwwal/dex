# Typography Divergence

Use when type is part of the design question, especially for dense dashboards, editorial/product hybrids, onboarding, or brand expression.

## Typography strategies

### 1. Single-family product system

Best for dense app UI, tools, dashboards, consistency.

### 2. Sans + mono

Best for product UI with technical data, logs, IDs, code, numbers.

### 3. Display + body

Best for brand surfaces, editorial surfaces, onboarding, memorable moments.

### 4. Serif + sans

Best for editorial, trust, long-form, reflective tone.

### 5. Condensed + neutral

Best for dense labels, dashboards, tables, nav-heavy surfaces.

### 6. Variable font strategy

Best when the interface needs expressive variation without multiple font files.

## Typography quality gates

- Is the hierarchy visible at a glance?
- Are headings, body, labels, and captions distinct?
- Is body text readable?
- Are line lengths comfortable?
- Are weights sufficiently different?
- Are there too many typefaces?
- Does type fit product vs brand register?
- Does it survive localization and larger text settings?

## Default stance

For product dashboards, default to a single tuned family or sans + mono. Decorative display fonts are usually wrong unless the surface is brand/editorial or a deliberately memorable moment.
