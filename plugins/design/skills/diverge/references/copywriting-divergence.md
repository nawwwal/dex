# Copywriting Divergence

Use when words are the design problem: labels, CTAs, onboarding text, empty states, errors, confirmations, feature education, notifications, and product voice.

## Copy roles

- Orient: where am I?
- Explain: what does this mean?
- Instruct: what should I do?
- Label: what is this?
- Warn: what could go wrong?
- Reassure: am I safe?
- Confirm: what happened?
- Recover: how do I fix it?
- Persuade: why act now?
- Teach: how do I get better?
- Celebrate: what did I accomplish?

## Copy divergence axes

1. Direct vs explanatory
2. Calm vs urgent
3. Expert vs beginner
4. Functional vs expressive
5. Action-first vs consequence-first
6. Short vs specific
7. Reassuring vs challenging
8. Human voice vs system voice
9. Instruction vs invitation
10. Error as blame vs error as recovery

## Copy quality gates

- Does the copy tell the user what changed?
- Does it tell the user what to do next?
- Does it name consequences?
- Does it remove ambiguity?
- Does it match the user's mental state?
- Does it avoid repeating the heading?
- Does it avoid generic AI filler?
- Does the CTA describe outcome, not vague action?
- Is humor appropriate to context?
- Can it localize without breaking?

## CTA rules

Bad:

- Submit
- Continue
- OK
- Confirm

Better:

- Save changes
- Reconnect Slack
- Delete 23 files
- Pause agent
- Approve new permission
- Retry sync
- Test connection
- Compare plans

## Error copy formula

Use:

```text
What happened -> why -> what to do next -> whether data is safe
```

Example:

"We could not reconnect Slack because the permission request expired. Your agents are still paused. Try reconnecting again."

## Empty state formula

Use:

```text
Why empty -> what value appears here -> next best action
```

Example:

"No connector issues right now. If a connector starts affecting an agent, it will appear here with the fix."
