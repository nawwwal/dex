# User Education

Principle:
Do not use education UI to compensate for unclear UX.

## Education hierarchy

1. Make the interface self-evident.
2. Use labels and affordances.
3. Use empty states for first-run orientation.
4. Use inline helper text only where labels cannot carry meaning.
5. Use targeted, non-blocking tips for users who already attempted the slower path.
6. Teach advanced shortcuts only after users have context.
7. Let valuable interactions spread socially where relevant.

## Education patterns

- Empty state orientation
- Inline helper
- Progressive hint
- One-time toast
- Shortcut reveal
- Learn-by-doing sandbox
- Example data
- Guided first success
- Contextual checklist
- Documentation link
- Social discovery

## Education quality gates

- Is this teaching something the UI should make obvious?
- Is it blocking the user?
- Is it targeted to the right user?
- Does it appear at the right moment?
- Does it save time for an existing behavior?
- Can the user ignore it?
- Does it repeat annoyingly?
- Does it help later adopters without annoying early adopters?
