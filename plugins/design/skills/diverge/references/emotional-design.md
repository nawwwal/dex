# Emotional Design

Use emotional design as product branding applied inside interaction, not decoration.

## Three layers

### Visceral

First impression.

Created through:

- Color
- Type
- Shape
- Density
- Motion
- Sound
- Imagery
- Perceived polish

### Behavioral

Feeling during use.

Created through:

- Speed
- Clarity
- Feedback
- Control
- Error recovery
- State visibility
- Predictability
- Reduction of effort

### Reflective

Meaning after use.

Created through:

- Pride
- Trust
- Memory
- Identity
- Progress history
- Shareability
- Audit trail
- Sense of mastery

## Emotional targets

Relief:
- Reduce choices.
- Show one next action.
- Confirm safety.
- Use calm language.
- Remove visual noise.

Confidence:
- Show evidence.
- Preview consequence.
- Use specific copy.
- Make reversibility clear.
- Show system reasoning.

Urgency:
- Name severity.
- Name deadline.
- Make action obvious.
- Remove exploration.
- Avoid cute copy.

Calm:
- Stable layout.
- Muted palette.
- Low motion.
- Clear status.
- No unnecessary interruption.

Mastery:
- Show progress.
- Reveal shortcuts.
- Support advanced use.
- Give feedback that teaches.

Trust:
- Show source, history, and reasoning.
- Use audit logs.
- Use consistent states.
- Avoid magic.

Pride:
- Recognize meaningful completion.
- Show artifact created.
- Make progress visible.
- Avoid celebrating trivial clicks.

Curiosity:
- Reveal deeper layers.
- Offer safe exploration.
- Let users inspect without commitment.

## Emotional quality gates

- What emotion exists before use?
- What emotion should exist during use?
- What emotion should remain after use?
- Which UI decisions create that emotion?
- Is the emotion appropriate to context?
- Does emotional design help the task?
- Would it still feel good after repeated use?
