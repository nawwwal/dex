# Test 1: Connector health dashboard

## Prompt

```md
/diverge "Diverge on a connector health dashboard for agents. Health states include expired auth, rate limit, API down, scope changed, schema mismatch, setup incomplete, not connected. Current anti-pattern is app grid with alerts."
```

## Bad output

- Quiet Sentinel
- Health Weather
- Permissions Diff Room
- Connector Black Box
- Garden of Integrations

## Good output

- Agent-risk board
- Issue repair queue
- Dependency map
- Permission/schema review
- Exception inbox
- Connector inventory with impact overlay

## Expected

The skill should not ask the full intake because the prompt includes surface, objects, states, and anti-pattern.

It should include:

- Brief
- Assumption Ledger
- Product Model
- State/action matrix
- Layer Diagnosis
- Obvious Baseline
- Banned Bad Directions
- Divergence Strategy
- 5-7 directions
- Comparison
- Recommendation
- Prototype slices
- Handoff Blueprint

Each direction must include:

- Product bet
- Layers changed
- What stays constant
- User job
- Core mechanic
- Screen anatomy
- Information hierarchy
- Interaction loop
- Copy examples
- Layout execution
- Typography execution
- Color execution
- Motion and feedback
- Emotional design
- Persuasive design
- States
- Data required
- Component requirements
- Edge cases
- Tradeoff
- Prototype slice
- Story mode
- Handoff notes

## Pass criteria

- Altitude is Mixed or State-handling-level with Product/UX layers.
- API down does not show the same action path as expired auth.
- Scope changed uses review/approval or permission diff.
- Schema mismatch includes mapping or compatibility evidence.
- Copy examples include state-specific CTA or error text.
- Handoff notes include objects, states, events, data, permissions, accessibility, and QA.
- Obvious baseline is judged seriously rather than dismissed for novelty.
