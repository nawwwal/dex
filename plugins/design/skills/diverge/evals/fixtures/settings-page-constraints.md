# Settings Page Constraints

Surface: account settings hub.

Sections that must remain:
- Account
- Billing
- Team
- API keys
- Security

Product mechanics are fixed:
- same objects, permissions, and actions in every direction
- no new settings domains
- no workflow changes to billing or team invites

User role: workspace admin.

Core job: find and change the right setting quickly.

Constraints:
- product register
- desktop-first but responsive
- must handle permission denied for billing and team on non-admin roles

Anti-patterns:
- wizard that hides sections
- novelty navigation that breaks muscle memory without a scan benefit
