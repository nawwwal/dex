## SynthID Watermark

All images generated by Gemini models include a SynthID digital watermark.
This is automatic, non-removable, and cannot be opted out of.

## Image Input for Editing

When `--input-image` is passed, the script runs in editing mode. Supported MIME types:

- `image/png`
- `image/jpeg`
- `image/webp`
- `image/heic`
- `image/heif`

Optional `resolution` field when sending image input via Interactions API:
`low` | `medium` | `high` | `ultra_high`

The script uses the `generateContent` surface (not Interactions API) for simplicity.
PIL handles format detection automatically.

## Thinking Config

`thinking_level` controls reasoning depth before generating:

- `minimal` (default) — fastest, standard quality
- `High` — slower, better for complex creative prompts, multi-element compositions

Thinking tokens are billed regardless of whether thoughts are surfaced.
The script always skips thought parts (`part.thought == True`) in output.

## Google Search Grounding

`--google-search` enables web search to ground the image in real-world visual context.
`--image-search` enables image search (Flash model only) — adds reference images from
the web to inform generation. Requires `--google-search` to also be set (handled
automatically by the script).

Attribution data from grounded responses must be displayed per Google Search ToS
if surfacing to end users.

## Streaming (advanced)

Image chunks arrive as `ImageDelta` events in the Interactions API surface:

```json
{
  "event_type": "content.delta",
  "delta": {
    "type": "image",
    "data": "<base64_chunk>",
    "mime_type": "image/png"
  },
  "index": 1
}
```

The `generate.py` script does not use streaming. For streaming use, write inline code
using `client.interactions.create(model=..., input=..., stream=True)`.
