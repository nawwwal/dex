# Vault Mode

Use when the investigation is centered on notes, docs, memory scaffolds, journals, decisions, patterns, or knowledge hygiene.

## Evidence preference

Prefer:

- docs and markdown files
- journal or log artifacts
- decision files
- pattern libraries and templates

## Mode-specific lenses

Choose from these after the permanent lenses are covered.

### References and link graph

- Focus on: what points here, what is orphaned, and which artifacts depend on this knowledge
- Do not duplicate: dependency mapping at code or system level

### History and drift

- Focus on: stale guidance, conflicting historical advice, and how the topic evolved over time
- Do not duplicate: blind-spot analysis

### Confusion hotspots

- Focus on: repeated ambiguity, duplicated guidance, unclear ownership, and places a new reader would stumble
- Do not duplicate: actionability; stay diagnostic

### Missing documentation

- Focus on: assumptions that are relied on but not written down
- Do not duplicate: the devil's advocate lens

## Good fit examples

- memory hygiene reviews
- scaffolding audits
- inconsistency hunts across docs
- pattern-library archaeology
