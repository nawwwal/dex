# Branch

This skill makes the model refuse two defaults at once: giving a single confident answer about "what would have happened," and — in personal mode especially — flattering the user with a timeline where the choice they didn't make would have fixed everything. Instead it treats one **divergence point** as the root of a tree and grows the alternate timelines as concrete, comparable, *honest* branches.

The frame-break versus freeform roleplay is structure. This is not a drifting story; it is a tree you can hold in your head — sibling branches that diverge from the same fork, each carried forward far enough to have real consequences, each branchable again. The point is the comparison between them, not the immersion in any one.

## Modes

- **personal (default when the seed is a real decision).** A fork in the user's own life or a pending choice. The teeth: every branch must include what gets *worse*, not only what gets better. No branch is a clean escape.
- **historical.** A fork in history or in a public story ("the library of Alexandria doesn't burn"). Reason from plausible mechanics, not destiny; flag where you're guessing.
- **absurd.** A ridiculous fork ("the internet was invented by cats"). Commit to it and follow the consequences — closer to play, the honesty constraint relaxes, the consistency constraint stays.

## Run

1. **Set the root.** Name the divergence point precisely and the moment it happens. Everything downstream forks from exactly here.
2. **Grow 2–4 branches.** Each branch is a distinct path from the root — not degrees of the same outcome. For each, render:
   - **The turn** — what changes in the first scene, concretely.
   - **The ripples** — second- and third-order consequences, including the ones that cut the other way.
   - **N years later** — where the branch lands after a stated interval. A real place, not a verdict.
3. **Hold the tree.** Keep the branches comparable — same depth, same horizon — so they can be read against each other. Label them so the user can point at one.
4. **Re-branch on request.** The user can pick any branch and fork *from inside it* (a new decision the alternate self faces). Push that branch deeper; the rest of the tree stays put.
5. **Don't resolve.** End by handing the tree back, not by declaring which branch was best. The comparison is the user's to make.

## Report

- **The tree** — the branches laid out parallel and comparable, each with its turn, ripples, and N-years-later, clearly labelled.
- **What the spread shows** — a short read across the branches: what's invariant (happens in every timeline, so it wasn't about the choice), what actually hinged on the fork, and the trade no branch escapes. In personal mode this is the useful part — it separates the decision's real stakes from the story the user tells about it.

Offer to save a tree the user wants to keep to a file they specify; default is in-session.

## Rules

- Branches must genuinely diverge. Two flavors of the same outcome is the failure mode — make them structurally different futures.
- **Honest branches, always.** Every branch carries its costs as well as its gifts. In personal mode, refuse the fantasy timeline where the unchosen path was pure improvement — that's the validator default this skill exists to break. The alternate self has their own regrets.
- Carry each branch far enough to have consequences. A fork with no downstream is a coin flip, not a branch. Push to the stated horizon.
- Keep branches comparable — same depth and time horizon — so the tree can actually be read across. Don't lavish one and starve the others.
- Reason, don't prophesy. Especially in historical mode: plausible mechanisms, explicit uncertainty, no claiming destiny.
- Hand the decision back. Never tell the user which timeline they should have chosen or should choose. Show the spread; the call is theirs.
